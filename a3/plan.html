Technologies and Architecture

Python Flask v1.1.2

Flask is a lightweight microframework that makes the development of a model/view/controller application easier. It allows the developer to specify models that incoming data must adhere to. It also has the ability to render templates using the built-in Jinja templating engine.

We have chosen Python Flask as it has less of a learning curve than other frameworks such as Django and this will aid the speed of our development.

MongoDB Community Server Edition v4.4.1(current)

MongoDB is a noSQL database system that can return JSON and doesn't require a strict definition of its tables (referred to as collections). MongoDB does not have relationships in the same way as a RDBMS (Relational Database Management System) instead it uses references (embedded) to other collections within the collection itself.

Pymongo v3.11.0

Pymongo is a python package that assist the Python programming language in interacting with MongoDB. It includes many tools and shortcut functions and includes a native python driver for Python/MongoDB.

Mongoengine

Mongoengine is a ODM (Object-Document Mapper) that serves as an abstraction layer over the Mongo database and pymongo driver. It simplifies the code you need to write for defining classes, inserting collections and performing database operations.

Pip v20.2.2

In order to install the Python packages that we require (Flask, pymongo) we may need to use Pip, which stands for Pip Installs Packages. Pip was developed by the MIT and allows the seamless addition of many python modules to a local system using the terminal/command prompt.

Amazon Web Services: EC2 (Elastic Compute Cloud)

AWS allows you to rent virtual computers in order to host your application. Using a free tier instance we plan on deploying the backend (Flask) part of our application using Amazon's EC2. Using an Amazon Machine Image (Ubuntu Server) we will be able to run Flask and Amazon will scale up instances of the application as required.

Amazon Web Services: S3 (Simple Storage Service)

For our front end assets (.png, .jpg) we plan on using Amazon's S3 service to store these objects that will be acessable via a web interface. This will provide fast delivery of our static assets that will be served by the Flask application.

Github

Github is website that provides a graphical user interface for the commands line tool Git. Git will allow us to control our development workflow, and create different branches for development. It also allows for easier collaboration, and the ability to revert our codebase. Once we have a working codebase we can merge our changes into once source.
